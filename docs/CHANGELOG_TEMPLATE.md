# CHANGELOG Template

  

This is the template for adding new entries to `CHANGELOG.md`.

New entries should be placed under the `## [Unreleased]` section (or a new version section) at the TOP of the file (below the header).

  

---

  

### [TYPE]

  

#### [TITLE]

  

- **Date:** [YYYY-MM-DD]

- **Commits:** `[SHORT_HASH]`

- [Change description 1]

- [Change description 2]

  

---

  

## Types of CHANGELOG:

- **Features**: For new features.

- **Refactoring**: For code CHANGELOG that neither fix a bug nor add a feature.

- **Bug Fixes**: For bug fixes.

- **Documentation**: For documentation CHANGELOG.

- **Style**: For CHANGELOG that do not affect the meaning of the code (white-space, formatting, missing semi-colons, etc).

- **Chores**: For maintenance tasks, dependency updates, etc.
